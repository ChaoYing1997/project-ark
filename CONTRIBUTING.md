# Contributing to Project Ark (方舟計畫)

首先，非常感謝你考慮為「方舟計畫」貢獻心力！正是像你一樣的熱心夥伴，讓這個開源專案充滿了生命力。

我們歡迎任何形式的貢獻，包括但不限於：
* 提交程式碼 (Pull Requests)
* 回報問題 (Bug Reports)
* 提出新功能建議 (Feature Requests)
* 完善文件
* 分享與推廣專案

## 回報問題與建議

我們使用 **GitHub Issues** 來追蹤所有問題與建議。在提交前，請先搜尋是否已有類似的議題存在。
* **[回報 Bug](https://github.com/ChaoYing1997/project-ark/issues/new?assignees=&labels=bug&template=bug_report.md&title=)**
* **[提出功能建議](https://github.com/ChaoYing1997/project-ark/issues/new?assignees=&labels=enhancement&template=feature_request.md&title=)**

## 程式碼貢獻流程 (Pull Request Workflow)

1.  **Fork a Repo:**
    點擊本 Repo 右上角的 "Fork" 按鈕，將專案複製一份到你自己的 GitHub 帳號下。

2.  **Clone your Fork:**
    將你 Fork 的 Repo 複製到你的電腦本地。
    ```bash
    git clone [https://github.com/Your-Username/project-ark.git](https://github.com/Your-Username/project-ark.git)
    ```

3.  **Create a Branch:**
    為你準備修改的功能或修復的 Bug，建立一個新的分支 (Branch)。分支名稱應該簡潔且能描述其目的。
    ```bash
    git checkout -b feature/amazing-new-feature
    ```

4.  **Make your Changes:**
    進行程式碼的修改與開發。

5.  **Commit your Changes:**
    提交你的變更，並撰寫清晰的 Commit message。
    ```bash
    git commit -m "feat: Add some amazing new feature"
    ```

6.  **Push to your Fork:**
    將你的分支推送到你 Fork 的遠端 Repo。
    ```bash
    git push origin feature/amazing-new-feature
    ```

7.  **Submit a Pull Request (PR):**
    回到你 Fork 的 GitHub 頁面，點擊 "New pull request" 按鈕，提交 PR 到主專案的 `main` 分支。請在 PR 的描述中詳細說明你所做的變更。

專案維護者會盡快審核你的 PR，並提供回饋。

## 行為準則

我們期望所有貢獻者都能遵守我們的 **[行為準則 (Code of Conduct)](CODE_OF_CONDUCT.md)**，共同維護一個友善、正向的協作環境。

再次感謝你的貢獻！
